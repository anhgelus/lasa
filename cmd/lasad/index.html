Lasa

Lasa is a stateless proxy that generates a RSS or an Atom feed from a Standard.site publication. It automatically fetches the publication and its linked documents from the PDS and returns a feed.

How to use it?

You can list the publications of example.org by going to /example.org. Each publication has a unique record key. This key is used to generate its RSS or its Atom feed.

If example.org has a publication with fooBar as record key, its RSS feed is: /example.org/fooBar/rss, and its Atom feed is: /example.org/fooBar/atom.

Some publications cannot have a RSS or an Atom feed, because they don't have given enough information to generate one. If the RSS feed doesn't work, try the Atom one.

Examples:

  • pckt - Dev Journal
  • Daniel's Leaflets

How does it works?

When you request a feed, Lasa parses the handle (or the did) of the user and the record key. Then, it looks in the cache if it already has generated this feed in the past hour. If it did, it returns this. If it didn't, it fetches the publication and the documents to generate the feed.

Lasa is open-source: check out the source code hosted on Tangled, an ATProto forge.
